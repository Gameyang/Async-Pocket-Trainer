# Async Pocket Trainer

서버 없이 정적 웹으로 동작하는 비실시간 포켓몬풍 자동 전투 로그라이크 게임입니다.

플레이어는 훈련사가 되어 야생 몬스터를 포획하고, 더 좋은 개체로 팀을 계속 교체하며, 같은 진행도에 도달한 다른 유저들의 팀 기록과 비동기 대전을 진행합니다.

> 핵심 목표는 “레벨업으로 강해지는 게임”이 아니라,  
> **더 좋은 야생 포켓몬을 발견하고 팀을 교체하면서 강해지는 게임**입니다.

---

## 게임 콘셉트

이 게임은 `Pokerogue`처럼 전투 중심으로 진행됩니다.

플레이어는 매 웨이브마다 다음 상황을 반복합니다.

1. 야생 포켓몬과 전투
2. 승리 후 포획 기회 획득
3. 기존 팀과 비교해서 더 좋은 포켓몬이면 교체
4. 보상 재화로 회복, 정비, 포켓볼 구매
5. 일정 웨이브마다 다른 유저의 팀 기록과 비동기 대전

플레이어의 포켓몬은 전통적인 레벨업 방식으로 성장하지 않습니다.

대신, 디아블로식 아이템 파밍처럼 야생 포켓몬을 계속 만나고, 그중에서 더 좋은 스탯과 기술 구성을 가진 포켓몬을 찾아 팀에 편성하는 방식입니다.

---

## 핵심 특징

- 서버 없는 정적 HTML/WebGL 게임
- GitHub Pages로 배포 가능
- 유저 진행도는 브라우저 로컬 저장소에 저장
- 비동기 PvP 지원
- 같은 웨이브 진행도의 다른 유저 팀 데이터와 대전
- 초기 151마리 포켓몬 기반 전투 시스템
- 포켓몬마다 정해진 스킬 풀에서 랜덤 기술 구성
- 레벨업이 아닌 포획과 교체 중심 성장
- 강력한 팀 조합을 찾는 로그라이크식 반복 플레이

```text
게임 시작

  ↓

  ↓

  ↓

  ↓

  ↓

  ↓

  ↓

  ↓

````

## 전투 시스템

전투 시스템은 초기 포켓몬 151마리 기반의 단순화된 룰을 사용합니다.

### 기본 전투 요소

* HP
* 공격
* 방어
* 특수
* 스피드
* 타입
* 기술
* 명중률
* 급소
* 상태 이상

### 전투 방식

전투는 턴제 자동 전투로 진행됩니다.

플레이어는 전투 중 직접 기술을 선택하지 않고, 팀 구성과 포켓몬의 기술 조합을 통해 승률을 높입니다.

```text
턴 시작
  ↓
양쪽 포켓몬의 스피드 비교
  ↓
빠른 포켓몬부터 기술 사용
  ↓
명중 판정
  ↓
타입 상성 계산
  ↓
데미지 적용
  ↓
상태 이상 / 추가 효과 처리
  ↓
기절 여부 확인
  ↓
다음 포켓몬 등장
```

## 포켓몬 생성 방식

야생 포켓몬은 등장할 때마다 고정된 종족값을 기반으로 하지만, 개체별 스탯과 기술은 랜덤하게 결정됩니다.

### 예시

```json
{
  "species": "Pikachu",
  "type": ["Electric"],
  "baseStats": {
    "hp": 35,
    "attack": 55,
    "defense": 40,
    "special": 50,
    "speed": 90
  },
  "movePool": [
    "Thunder Shock",
    "Quick Attack",
    "Thunder Wave",
    "Double Team",
    "Slam",
    "Thunderbolt"
  ]
}
```

실제 야생 포켓몬 생성 시:

```json
{
  "species": "Pikachu",
  "stats": {
    "hp": 41,
    "attack": 58,
    "defense": 36,
    "special": 54,
    "speed": 96
  },
  "moves": [
    "Thunder Shock",
    "Quick Attack",
    "Double Team",
    "Thunderbolt"
  ],
  "rarityScore": 82
}
```

같은 피카츄라도 스탯과 기술 구성이 다르기 때문에, 플레이어는 계속해서 더 좋은 개체를 찾게 됩니다.

## 성장 방식

이 게임에는 일반적인 레벨업 시스템이 없습니다.

플레이어의 팀은 다음 방식으로 강해집니다.

* 더 높은 스탯의 포켓몬 포획
* 더 좋은 기술 조합의 포켓몬 포획
* 현재 팀과 신규 포켓몬 비교
* 약한 포켓몬을 방출하고 강한 포켓몬으로 교체
* 웨이브가 높아질수록 더 강한 야생 포켓몬 등장

즉, 포켓몬은 캐릭터라기보다 “아이템 드랍”에 가깝습니다.

```text
좋은 검을 얻으면 기존 검을 버리는 것처럼,
좋은 포켓몬을 얻으면 기존 포켓몬을 교체합니다.
```

---

## 포획 시스템

야생 포켓몬과의 전투에서 승리하면 포획 기회가 주어집니다.

포획에는 포켓볼이 필요합니다.

### 포획 확률 요소

* 야생 포켓몬의 희귀도
* 현재 웨이브
* 포켓몬의 남은 HP
* 포켓볼 종류
* 상태 이상 여부
* 플레이어 보너스 효과

### 포획 후 선택

포획에 성공하면 기존 팀과 비교할 수 있습니다.

```text
새 포켓몬을 팀에 추가하시겠습니까?

현재 포켓몬:
- Charmeleon
- HP 72
- Attack 68
- Defense 55
- Special 63
- Speed 74
- Moves: Ember, Scratch, Leer, Fire Spin

새 포켓몬:
- Growlithe
- HP 76
- Attack 80
- Defense 48
- Special 62
- Speed 70
- Moves: Ember, Bite, Roar, Flamethrower
```

플레이어는 새 포켓몬을 채용하거나 포기할 수 있습니다.

## 재화 시스템

전투에서 승리하면 재화를 얻습니다.

재화는 다음 용도로 사용됩니다.

* 팀 전체 회복
* 기절한 포켓몬 부활
* 상태 이상 치료
* 포켓볼 구매
* 특수 포켓볼 구매
* 다음 전투 전 정비
* 리롤 또는 스카우트 기능

### 예시 상점

| 항목         | 설명                 |
| ---------- | ------------------ |
| Potion     | 포켓몬 1마리 HP 회복      |
| Full Heal  | 상태 이상 치료           |
| Revive     | 기절한 포켓몬 부활         |
| Poké Ball  | 기본 포획 도구           |
| Great Ball | 높은 포획 확률           |
| Team Rest  | 팀 전체 회복            |
| Scout      | 다음 야생 포켓몬 후보 미리 보기 |

## 비동기 트레이너 대전

이 게임의 PvP는 실시간 대전이 아닙니다.

다른 유저가 특정 웨이브에 도달했을 때의 팀 상태가 저장되고, 이후 같은 웨이브에 도달한 다른 유저가 그 팀 데이터를 받아 전투합니다.

### 대전 발생 웨이브

```text
5, 10, 15, 20, 25...
```

예를 들어, 어떤 유저가 10웨이브에 도달했을 때 팀 데이터가 기록됩니다.

```json
{
  "wave": 10,
  "trainerName": "BlueTrainer",
  "team": [
    {
      "species": "Ivysaur",
      "stats": {
        "hp": 81,
        "attack": 65,
        "defense": 67,
        "special": 82,
        "speed": 61
      },
      "moves": ["Vine Whip", "Sleep Powder", "Razor Leaf", "Leech Seed"]
    }
  ]
}
```

다른 유저가 10웨이브에 도달하면 이 기록 중 하나를 다운로드하여 대전 상대로 사용합니다.

---

## 비동기 PvP 목표

비동기 대전은 단순한 체크포인트 보스 역할을 합니다.

플레이어는 다음을 목표로 합니다.

* 같은 진행도의 다른 유저보다 강한 팀 만들기
* 좋은 스탯과 기술 조합 찾기
* 특정 타입 조합 카운터 준비
* 연승 기록 갱신
* 최종적으로 가장 강한 팀 조합 발견

시간이 지나면 자연스럽게 같은 웨이브 구간에서 강력한 조합들이 선별됩니다.

## 데이터 저장 구조

이 프로젝트는 전용 게임 서버를 사용하지 않습니다.

### 로컬 데이터

플레이어 개인 진행도는 브라우저 로컬 저장소에 저장합니다.

사용 후보:

* `localStorage`
* `IndexedDB`

저장 데이터:

```json
{
  "playerId": "local-random-id",
  "trainerName": "Player",
  "currentWave": 12,
  "money": 340,
  "balls": {
    "pokeBall": 5,
    "greatBall": 1
  },
  "team": [],
  "runHistory": []
}
```

---

## 백엔드 없는 비동기 데이터 공유

이 게임은 서버를 직접 운영하지 않는 것을 목표로 합니다.

대신 Google Calendar를 단순 저장소처럼 사용합니다.

### 기본 아이디어

각 유저가 특정 웨이브에 도달하면 현재 팀 상태를 Google Calendar 이벤트 형태로 기록합니다.

다른 유저는 비동기적으로 Calendar 이벤트를 읽어와 대전 상대 데이터로 사용합니다.

```text
유저 A: 10웨이브 도달
  ↓
팀 데이터 JSON 생성
  ↓
Google Calendar 이벤트로 업로드
  ↓
유저 B: 10웨이브 도달
  ↓
10웨이브 팀 데이터 목록 다운로드
  ↓
랜덤으로 하나 선택
  ↓
비동기 트레이너 대전
```

## 핵심 재미

이 게임의 재미는 다음 질문에서 나옵니다.

> “이번에 잡은 포켓몬이 내 팀의 기존 포켓몬보다 더 좋은가?”

플레이어는 같은 포켓몬이라도 더 좋은 스탯, 더 좋은 기술, 더 좋은 타입 조합을 기대하며 계속 전투를 진행합니다.

그리고 그렇게 완성한 팀이 같은 웨이브의 다른 유저 팀과 겨루면서, 자연스럽게 강한 조합이 선별됩니다.

## 라이선스 및 주의사항

이 프로젝트는 포켓몬에서 영감을 받은 팬 게임 또는 프로토타입 아이디어입니다.

실제 공개 배포 시에는 다음 사항을 주의해야 합니다.

* Pokémon, 포켓몬 이름, 이미지, 기술명 등은 Nintendo, Game Freak, The Pokémon Company의 지식재산일 수 있습니다.
* 공개 서비스 목적이라면 오리지널 몬스터, 오리지널 타입, 오리지널 기술명으로 대체하는 것을 권장합니다.
* 이 README는 게임 시스템 설계 초안이며, 상업적 사용을 보장하지 않습니다.
