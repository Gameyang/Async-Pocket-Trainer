# Async Pocket Trainer 비주얼 기획서

## 1. 목적

이 문서는 `docs/game-direction-plan.md`의 게임 방향을 기준으로 MVP 화면 비주얼과 상호작용을 정의한다. 게임 화면은 크게 세 가지 씬으로 구성한다.

1. 포켓몬 도감 스타터 선택 씬
2. 상점/팀정비 씬
3. 전투 씬

UI 언어는 한국어로 유지하고, 포켓몬 이미지는 기존 `src/resources/pokemon/0001.webp`-`0151.webp` 자산을 사용한다. 전투 화면 상단은 현재 구현된 `class="battle-frame"` 계열의 포켓몬 원작풍 HUD와 배틀 패널을 유지한다.

## 2. 공통 비주얼 원칙

- 전체 화면은 모바일 세로 화면을 1차 기준으로 설계한다.
- 카드, 버튼, 패널은 현재 앱의 8px 이하 라운드와 굵은 외곽선 스타일을 유지한다.
- 중요한 조작은 화면 하단 또는 선택된 카드 위에 직접 배치한다.
- 전투 중에는 조작 가능한 UI와 읽기 전용 UI를 명확히 구분한다.
- 모달성 선택 상태에서는 반투명 블랙 마스크를 사용해 현재 선택 대상만 강조한다.
- 스크롤 가능한 영역은 한 화면에 하나만 주도권을 갖게 해 의도치 않은 스크롤 충돌을 줄인다.
- HP, 재화, 선택 가능 슬롯처럼 상태가 바뀌는 요소는 색, 외곽선, 비활성 상태를 함께 사용해 구분한다.

## 3. 전체 씬 흐름

```text
도감 스타터 선택
  -> 상점/팀정비
  -> 전투
      -> 일반 야생전 승리: 포켓볼 사용 탭
          -> 포획 결과 처리
          -> 상점/팀정비
      -> 일반 야생전 패배
          -> 도감 스타터 선택
      -> 트레이너전 승리
          -> 팀 이름/인사말 편집
          -> 상점/팀정비
      -> 트레이너전 패배
          -> 팀 이름 편집
          -> 도감 스타터 선택
```

## 4. 도감 스타터 선택 씬

### 화면 목적

새 모험을 시작할 스타터 포켓몬을 고르는 화면이다. 도감은 1세대 151종 전체를 항상 보여주되, 언락 여부에 따라 정보 노출 수준을 다르게 한다.

### 레이아웃

- 화면 전체는 세로 스크롤 가능한 도감 그리드다.
- 그리드는 151개 포켓몬 카드를 도감 번호 순서로 배치한다.
- 모바일 기본은 3열 그리드를 사용하고, 넓은 화면에서는 4-6열까지 확장할 수 있다.
- 각 카드는 고정 높이를 사용해 언락/잠금 상태 전환 시 레이아웃이 흔들리지 않게 한다.

### 도감 카드 상태

언락된 포켓몬:

- 정상 포켓몬 이미지 표시
- 도감 번호 표시
- 포켓몬 이름 표시
- 스타터 선택 가능 상태 표시

잠긴 포켓몬:

- 포켓몬 실루엣만 표시
- 이름은 `???`로 표시
- 도감 번호는 표시한다.
- 스타터 선택은 불가능하다.

언락 조건:

- 포켓몬이 한 번이라도 팀에 합류하면 해당 종이 도감에 언락된다.
- 첫 접속 기본 언락은 기존 설계대로 `001`, `004`, `007`이다.

### 선택 인터랙션

1. 유저가 언락된 도감 카드를 누른다.
2. 선택된 카드 위에 `선택` 버튼과 `취소` 버튼을 표시한다.
3. 선택된 카드를 제외한 전체 화면에는 블랙 반투명 마스크를 덮는다.
4. 마스크가 깔린 동안 다른 카드 클릭과 도감 스크롤은 비활성화한다.
5. `취소`를 누르면 선택 상태를 해제하고 기존 스크롤 위치를 유지한다.
6. `선택`을 누르면 해당 포켓몬으로 새 모험을 시작하고 상점/팀정비 씬으로 이동한다.

### 비주얼 상태

- 선택된 카드는 마스크보다 위 레이어에 두고, 외곽선과 밝기 상승으로 강조한다.
- 잠긴 카드는 채도와 명도를 낮추고, 실루엣 이미지에는 `brightness(0)` 또는 동등한 마스크 효과를 사용한다.
- `선택` 버튼은 주 조작 색, `취소` 버튼은 보조 색으로 분리한다.
- 잠긴 카드를 눌렀을 때는 선택 버튼을 띄우지 않고 짧은 비활성 피드백만 준다.

## 5. 상점/팀정비 씬

### 화면 목적

전투 전후에 팀 상태를 확인하고, 회복/포켓볼/팀 편집 상품을 사용하는 메인 정비 화면이다.

### 전체 레이아웃

상점/팀정비 씬은 상단 팀 패널과 하단 상점 탭으로 나뉜다.

상단 팀 패널:

- 2열 x 3행의 6개 포켓몬 요약 카드 슬롯을 표시한다.
- 넓은 가로 화면에서는 3열 x 2행으로 전환할 수 있지만, 6슬롯 한 묶음이라는 인식은 유지한다.
- 빈 슬롯도 카드로 표시해 팀 최대치가 항상 보이게 한다.

하단 상점 탭:

- 가장 위에 재화 2종을 표시한다.
- 재화는 런 코인과 영구 보석으로 구분한다.
- 그 아래는 좌측 상품 리스트, 우측 상품 상세/조작 영역으로 나눈다.
- 모바일 폭이 좁을 때도 상품 리스트와 상세 영역은 같은 탭 안에서 함께 보이게 하되, 상세 영역이 조작 버튼을 가리지 않게 한다.

### 팀 요약 카드

각 포켓몬 슬롯 카드에는 다음 정보를 표시한다.

- 포켓몬 이미지
- 현재 HP와 최대 HP
- 레벨과 이름
- 공격기 이름
- 보조기 이름
- `hp`, `atk`, `def`, `spa`, `spd`, `spe` 6개 스탯 수치

빈 슬롯:

- 비어 있음 상태를 표시한다.
- 상품 적용 대상 선택 중에는 빈 슬롯이 대상이 될 수 있는 경우에만 강조한다.

기절 포켓몬:

- 이미지 명도와 채도를 낮추고 HP를 0으로 표시한다.
- 회복 아이템 선택 중이면 적용 가능 대상으로 강조할 수 있다.

### 상품 리스트 정렬

상품은 아래 순서로 정렬한다.

1. 전투 필드로 이동
2. 포획 성공한 대기 포켓몬
3. 상품 목록 갱신
4. HP 회복 상품
5. 포켓볼 상품
6. 포켓몬 판매/전투력 정렬 같은 팀 편집 상품

`전투 필드로 이동`과 포획 성공한 대기 포켓몬은 고정 상품으로 취급한다. 그 외 상품군은 현재 상점 상태를 만들 때 상품 타입 중 2-3종을 랜덤으로 노출한다.

### 상품 상세 영역

상품을 선택하면 우측 상세 영역에 다음 정보를 표시한다.

- 상품 이름
- 가격과 사용 재화
- 효과 설명
- 사용 조건
- 주요 조작 버튼

조작 버튼 예시:

- `전투 시작`
- `구매`
- `사용`
- `합류`
- `판매`
- `정렬`
- `취소`

`전투 필드로 이동` 상품은 중앙의 가장 중요한 조작으로 보여주며, 버튼 텍스트는 `전투 시작`으로 통일한다.

### 대상 선택형 상품 UX

HP 회복, 포켓몬 판매처럼 팀 슬롯 대상이 필요한 상품은 다음 순서로 처리한다.

1. 구매 또는 실행 전, 적용 가능한 대상이 존재하는지 먼저 검사한다.
2. 대상이 없으면 버튼을 비활성화하고 상세 영역에 이유를 표시한다.
3. 실행하면 비용을 먼저 지불하고 대상 선택 모드로 들어간다.
4. 상단 6개 슬롯 중 적용 가능한 슬롯만 선택 가능 상태로 강조한다.
5. 적용 불가능한 슬롯은 어둡게 처리하고 클릭을 막는다.
6. 유저가 하나의 슬롯을 선택하면 상품 효과를 적용한다.
7. `취소`를 누르면 대상 선택 모드를 종료하고 지불한 재화를 즉시 반환한다.

대상 선택 모드의 시각 효과:

- 상점 탭 상세 영역에는 현재 선택 중인 상품명과 `취소` 버튼을 고정 표시한다.
- 선택 가능한 슬롯에는 밝은 외곽선, 살짝 올라오는 위치 변화, 선택 가능 커서를 적용한다.
- 선택 불가능 슬롯은 반투명 마스크와 낮은 채도로 표시한다.
- 이 모드에서는 다른 상품 선택과 전투 시작을 비활성화한다.

### 포획 성공 대기 포켓몬 상품

- 팀에 빈 슬롯이 있으면 `합류` 버튼으로 즉시 팀에 넣을 수 있다.
- 팀이 가득 찬 경우 `합류` 버튼은 비활성화하고, 기존 포켓몬 판매로 슬롯을 비우도록 유도한다.
- 상점/팀정비 씬을 떠날 때까지 합류하지 않은 대기 포켓몬은 사라진다.

## 6. 전투 씬

### 화면 목적

자동 전투를 시청하고, 전투 중에는 팀 상태를 확인하며, 야생전 승리 후에는 포켓볼을 선택해 포획을 시도하는 화면이다.

### 전투 중 레이아웃

상단:

- 현재 구현된 `battle-frame` 구조를 유지한다.
- `hud enemy`, `hud player`, `sprite-slot enemy`, `sprite-slot player`, `message-box`를 중심으로 원작 포켓몬풍 전투 패널을 사용한다.
- 전투 결과 메시지는 `message-box`에 표시한다.

하단:

- 상점/팀정비 씬의 상단 팀 패널을 그대로 재사용한다.
- 6개 포켓몬 슬롯은 전투 중 읽기 전용이다.
- 전투 중 유저는 하단 팀 패널을 클릭할 수 없다.
- 현재 출전 중인 포켓몬 슬롯은 강조 표시한다.
- 쓰러진 포켓몬은 즉시 기절 상태로 표시한다.

### 야생전 종료 후 포켓볼 사용 탭

야생전에서 승리하면 하단 패널은 팀 상태 패널에서 포켓볼 사용 탭으로 전환된다.

- 이 탭은 상점/팀정비 씬의 하단 상점 탭 UI를 재사용한다.
- 좌측은 보유 포켓볼 리스트다.
- 우측은 선택한 포켓볼 상세 설명과 `사용` 버튼이다.
- 구매 UI가 아니라 보유 아이템 사용 UI이므로 가격 대신 보유 수량과 포획 예상 정보를 보여준다.
- 포켓볼을 사용하면 상단 `battle-frame` 안에서 던지기, 흔들림, 성공/실패 연출을 재생한다.
- 포획 성공 시 도감 언락과 팀 합류/대기 상품화를 처리한다.
- 포획 실패 시 상점/팀정비 씬으로 돌아간다.

### 트레이너전 종료 후 결과 편집

트레이너 배틀은 포획 탭을 표시하지 않는다. 결과가 나온 뒤 상단 `battle-frame`은 그대로 유지하고, 그 아래 또는 하단 패널 영역을 결과 편집 UI로 전환한다.

승리 시:

- 승리 메시지와 함께 축하 꽃가루 연출을 표시한다.
- 팀 이름 편집 입력을 표시한다.
- 인사말 편집 입력을 표시한다.
- 저장된 팀 이름과 인사말이 있으면 자동으로 불러와 기본값으로 채운다.
- 유저는 그대로 유지하거나 수정할 수 있다.
- 확인 후 상점/팀정비 씬으로 돌아가 모험을 이어간다.

패배 시:

- 패배 메시지를 표시한다.
- 팀 이름 편집 입력은 표시한다.
- 인사말 편집 입력은 생략한다.
- 저장된 팀 이름이 있으면 자동으로 불러와 기본값으로 채운다.
- 확인 후 도감 스타터 선택 씬으로 돌아가 새 스타터를 선택한다.

### 결과 편집 저장

- 팀 이름과 인사말은 `localStorage`의 `apt.player.v1`에 상시 저장한다.
- 트레이너전 결과 화면에 들어오면 저장된 값을 자동으로 불러온다.
- 승리 결과에서 인사말을 수정하면 이후 트레이너 스냅샷 업로드와 다음 결과 편집 기본값에 반영한다.
- 패배 결과에서는 인사말 입력을 받지 않지만, 기존 저장 인사말은 유지한다.

## 7. 주요 컴포넌트 재사용 계획

| UI | 사용 씬 | 재사용 방식 |
| --- | --- | --- |
| `battle-frame` | 전투 씬 | 상단 전투 패널로 고정 사용 |
| 팀 6슬롯 패널 | 상점/팀정비, 전투 | 상점에서는 조작 가능, 전투 중에는 읽기 전용 |
| 상품 리스트/상세 패널 | 상점/팀정비, 전투 후 포획 | 상점에서는 구매/편집, 포획에서는 보유 포켓볼 사용 |
| 대상 선택 슬롯 효과 | 상점/팀정비 | 회복, 판매 등 대상형 상품에 공통 적용 |
| 도감 카드 | 도감 스타터 선택 | 언락/잠금/선택 상태를 동일 카드에서 처리 |

## 8. 상태별 비주얼 체크리스트

도감:

- 151종 카드가 세로 스크롤로 모두 표시된다.
- 잠긴 포켓몬은 실루엣과 `???`로 표시된다.
- 언락 포켓몬은 이미지와 이름이 정상 표시된다.
- 카드 선택 시 선택 카드만 조작 가능하고, 나머지는 마스크 처리된다.
- 선택 상태에서 배경 클릭과 스크롤이 차단된다.

상점/팀정비:

- 6개 팀 슬롯이 항상 보인다.
- 각 포켓몬 카드에 이미지, HP, 레벨/이름, 2개 스킬, 6개 스탯이 보인다.
- 재화 2종이 상품 리스트 위에 표시된다.
- 상품 리스트는 지정 순서를 지킨다.
- 대상형 상품은 적용 가능 슬롯만 강조하고 취소 시 재화를 반환한다.
- 전투 시작 버튼은 명확하게 중앙 주 조작으로 보인다.

전투:

- 상단은 기존 `battle-frame` 구조와 원작풍 HUD를 유지한다.
- 전투 중 하단 팀 패널은 클릭되지 않는다.
- 전투 중 현재 출전 슬롯과 기절 슬롯이 구분된다.
- 야생전 승리 후 하단이 포켓볼 사용 탭으로 바뀐다.
- 트레이너전 결과 후 승리/패배에 따라 편집 입력 항목이 다르게 표시된다.
- 팀 이름과 인사말은 저장값을 자동으로 불러온다.

## 10. 오픈 결정 사항

- 잠긴 도감 카드 클릭 시 아무 반응을 주지 않을지, 짧은 잠금 피드백을 줄지 결정이 필요하다.
- 모바일에서 팀 카드에 6개 스탯까지 모두 표시할 때 카드 밀도가 높으므로 축약 표기와 폰트 크기 기준을 구현 시 확인해야 한다.
- 상품 타입 랜덤 2-3종의 정확한 타입 풀과 노출 개수는 밸런스 구현 단계에서 확정한다.
- 트레이너전 패배 시 팀 이름 편집을 스냅샷 업로드 전에 할지, 업로드 후 로컬 프로필만 갱신할지 구현 단계에서 저장 순서를 확정한다.
