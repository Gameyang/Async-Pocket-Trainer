# 151종 전투 스킬 구현 검토

검토일: 2026-05-14

범위는 `src/game/data/pokemonBattleRuntimeData.json`의 포켓몬 1-151번과 `lets-go-pikachu-lets-go-eevee` learnset 기준이다. 현재 게임은 이 learnset을 그대로 전부 쓰지 않고, `catalog.ts`에서 고정 위력의 물리/특수 공격기만 통과시킨 뒤 포켓몬별 최대 12개 후보로 자르고, 전투 진입 시 그중 최대 4개를 랜덤 배정한다.

## 결론

현재 전투는 플레이 가능한 간소화 모델이다. 타입 상성, 자속, 명중률, 물리/특수 분기, 급소, 랜덤 데미지, 주요 상태이상 일부는 실제 계산에 들어간다. 하지만 151종 풀스킬 구현 완료 상태는 아니다. 211개 learnset 기술 중 81개는 현재 게임에 들어오지 않고, 들어오는 114개 중 71개는 데미지는 계산되지만 고유 부가효과가 빠져 있다.

## 현재 파이프라인

| 영역 | 현재 구현 | 코드 |
| --- | --- | --- |
| 원본 데이터 | 151종, 211개 move, 18개 타입, LGPE learnset 포함 | `src/game/data/pokemonBattleRuntimeData.json` |
| 기술 필터 | `power > 0`이고 `physical` 또는 `special`인 기술만 `MoveDefinition`으로 변환 | `src/game/data/catalog.ts` |
| 포켓몬 기술 후보 | learnset 정렬 후 유효 공격기만 중복 제거, 포켓몬별 최대 12개만 보관 | `src/game/data/catalog.ts` |
| 전투 기술 배정 | 후보 12개를 셔플해 최대 4개 선택 | `src/game/creatureFactory.ts` |
| 기술 선택 AI | 현재 상대에게 예상 데미지가 가장 큰 기술 선택 | `src/game/battle/battleEngine.ts` |
| 데미지 계산 | 물리/특수 스탯, 기술 위력, 타입 상성, 자속, 명중률, 급소, 데미지 편차 반영 | `src/game/battle/damage.ts` |
| 상태이상 | 화상, 독, 마비, 잠듦, 얼음 일부 처리. 혼란, 트랩, 씨뿌리기 등은 없음 | `src/game/battle/battleEngine.ts` |

## 구현 순서 제안

1. 먼저 데이터 모델과 테스트 스냅샷을 만든다. 현재처럼 기술 효과가 silently drop 되는 구조를 막는 것이 우선이다.
2. 다음으로 랭크, 우선도, 흡수/반동, 풀죽음, 연속기를 구현한다. 이 묶음이 현재 부분 구현 81개 중 체감 누락이 가장 큰 영역이다.
3. 그 다음 status-only 기술을 movePool에 넣는다. 이때 기술 선택 AI가 데미지 기준 하나뿐이면 변화기를 절대 제대로 쓰지 못하므로 utility scoring이 같이 필요하다.
4. 마지막으로 본가식 예외 기술을 개별 effect handler로 연결한다. 151종 기준으로도 `손가락흔들기`, `변신`, `카운터`, `미러코트`, `대타출동`은 별도 설계가 필요하다.
