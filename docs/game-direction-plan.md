# Async Pocket Trainer 게임 설계 문서

## 1. 요약

`Async Pocket Trainer`는 1세대 포켓몬 151마리를 수집하고, 6개 팀 슬롯을 계속 교체하며, 자동 전투로 웨이브를 돌파하는 모바일 웹 게임이다. 핵심 방향은 `Backpack Battles`처럼 실시간 PvP가 아니라 다른 유저가 남긴 팀 스냅샷과 비동기로 싸우는 것이다.

- 클라이언트는 GitHub Pages 배포 가능한 Vite + TypeScript 정적 웹앱으로 유지한다.
- 포켓몬과 전투 규칙은 1세대 기준으로 시작하고, 기존 `src/resources/pokemon/0001.webp`-`0151.webp` 이미지를 사용한다.
- 전투 엔진은 Pokemon Showdown의 Gen1 데이터/로직을 참고한 로컬 경량 구현을 유지한다.
- 기본 게임 정보는 브라우저 `localStorage`에 저장하고, 트레이너전 유저 팀 스냅샷과 결과만 Google Sheets에 백업/동기화한다.
- MVP는 웨이브 진행, 야생 포켓몬 포획, 도감 언락 기반 스타터 선택, 보유 포켓몬 비교/교체, 6슬롯 팀 운용, 5웨이브 단위 비동기 트레이너전을 포함한다.

## 2. 핵심 재미

### 포켓몬을 아이템처럼 많이 얻는 게임

포켓몬은 희귀한 단일 캐릭터가 아니라 디아블로식 장비처럼 자주 얻고 비교하는 대상으로 설계한다. 플레이어는 새 포켓몬을 얻을 때마다 현재 팀과 비교하고, 6개 슬롯 중 무엇을 유지하거나 교체할지 빠르게 판단한다.

- 비교 기준: 종족, 타입, 레벨, HP, 공격/방어/특수/스피드, 전투강도, 공격기, 보조기, 현재 상태.
- 교체 판단: 즉시 팀 슬롯에 넣기, 임시보관 후 팀 편성하기, 기존 포켓몬을 판매/합성하기 중 하나를 선택한다.
- 플레이 감각: 좋은 포켓몬이 계속 떨어지고, 팀 조합이 웨이브마다 갱신되는 수집형 자동전투.

### 비동기 유저 팀과 싸우는 게임

플레이어는 일정 웨이브마다 야생 포켓몬이 아니라 다른 유저의 과거 팀 조합과 싸운다. 상대는 실시간 접속자가 아니라 Google Sheets에 저장된 팀 스냅샷이다.

- 5, 10, 15... 웨이브는 트레이너 배틀 웨이브다.
- 트레이너 배틀 상대는 같은 웨이브 구간에서 다른 유저가 사용했던 6슬롯 팀 상태다.
- 유저가 모험을 종료하거나 체크포인트에 도달하면 현재 팀 상태를 백엔드에 업로드한다.
- 다른 유저의 스냅샷은 이후 플레이어의 트레이너 배틀 콘텐츠가 된다.

## 3. MVP 게임 루프

1. 플레이어가 새 모험을 시작한다.
2. 게임은 시드와 `playerId`를 준비하고, 유저가 언락한 도감 목록에서 스타터 포켓몬을 선택한다.
3. 일반 웨이브에서는 야생 포켓몬 1마리와 자동 전투한다.
4. 야생 포켓몬의 HP가 0이 되면 포켓볼을 1회 던져 포획을 시도한다.
5. 포획 성공 시 빈 팀 슬롯이 있으면 자동 합류하고, 빈 슬롯이 없으면 임시보관 상태로 팀 편성 씬에 진입한다.
6. 5의 배수 웨이브에서는 야생 포켓몬 대신 비동기 트레이너 팀 스냅샷과 자동 전투한다.
7. 트레이너 배틀이 끝나면 승패와 관계없이 현재 팀 스냅샷을 Google Apps Script API로 전송한다.
8. 모든 아군 포켓몬 HP가 0이 되면 패배 처리하고, 스타터 선택 시작 화면으로 돌아간다.

## 4. 웨이브 설계

- 웨이브 번호는 1부터 시작한다.
- 일반 웨이브: `wave % 5 !== 0`
  - 야생 포켓몬 1마리가 등장한다.
  - 야생 포켓몬은 현재 웨이브를 기준으로 레벨 범위를 점진 상승시킨다.
  - 승리 후 포획 기회를 1회 제공한다.
- 트레이너 배틀 웨이브: `wave % 5 === 0`
  - 야생 포켓몬은 등장하지 않는다.
  - Apps Script API에서 해당 웨이브의 상대 팀 스냅샷을 요청한다.
  - 상대 스냅샷이 없거나 API가 실패하면 로컬 기본 트레이너 팀을 사용한다.

MVP 레벨 스케일링 기본값:

- 1-4웨이브: Lv.5-10
- 6-9웨이브: Lv.8-14
- 11-14웨이브: Lv.12-18
- 이후 5웨이브마다 최소/최대 레벨을 3씩 올린다.
- 레벨 상한은 MVP에서 Lv.50으로 제한한다.
- 웨이브 사이 HP는 유지한다.
- 웨이브 사이 화상, 독, 마비, 수면, 빙결 같은 이상 상태는 회복한다.
- 5의 배수 트레이너 배틀이 끝나면 플레이어 팀 전체 HP를 100% 회복한다.
- 트레이너 배틀에서 패배해 게임 오버가 되더라도 패배한 팀 스냅샷은 업로드한다.
- 야생전과 트레이너전 모두 모든 아군 포켓몬 HP가 0이 되면 패배한다.
- 야생전 패배는 스냅샷 업로드 없이 런 종료로 처리한다.
- 트레이너전 패배는 실패한 팀 스냅샷과 전투 결과를 Google Sheets에 동기화한 뒤 런 종료로 처리한다.

## 5. 전투 설계

### 기본 전투

- 자동 전투이며 플레이어는 전투 중 기술을 직접 선택하지 않는다.
- 현재 구현된 Gen1 기반 로컬 전투 엔진을 확장한다.
- MVP 전투는 PP를 소모하지 않는다.
- 기술 선택은 기존 방향을 유지한다.
  - 공격기 선택 확률 70%
  - 보조기 선택 확률 30%
  - 각 포켓몬은 레벨 이하에서 배운 공격기 1개와 보조기 1개를 시드 기반으로 보유한다.
  - 조건을 만족하는 기술이 없으면 타입별 기본 공격기와 공용 보조기 fallback을 사용한다.

### 팀 전투

MVP 최종 전투 단위는 1v1 단판이 아니라 6슬롯 팀 전투다.

- 각 팀은 최대 6마리 포켓몬을 가진다.
- 전투 시작 시 각 팀의 첫 번째 생존 포켓몬이 출전한다.
- 포켓몬이 쓰러지면 다음 생존 포켓몬이 자동 출전한다.
- 한쪽 팀의 모든 포켓몬이 쓰러지면 전투가 종료된다.
- 현재 1v1 엔진은 팀 전투의 단일 매치업 해석기로 사용한다.

### 포획

- 포획은 야생 포켓몬 전투에서만 발생한다.
- 야생 포켓몬 HP가 0이 된 뒤 포켓볼을 1회만 던진다.
- 트레이너 배틀에서는 포획이 발생하지 않는다.
- 포획 성공 시 해당 포켓몬은 HP 50% 상태로 회복된다.
- MVP 포획률은 포켓몬의 전투강도와 포켓볼 종류만으로 단순 계산한다.
  - `captureChance = pokeballPower / (pokeballPower + wildPower)`
  - `wildPower`는 포켓몬 레벨, 종족값 합, 현재 선택된 공격기/보조기 가중치를 합산한 값이다.
  - `pokeballPower`는 포켓볼 종류별 고정 값이다.
  - 최종 확률은 10%-90%로 제한한다.
- 포획 성공/실패는 시드 기반 RNG로 재현 가능해야 한다.

### 야생 포켓몬 등장

야생 포켓몬 등장 확률은 포획률과 별도 시스템으로 분리한다. MVP에서는 완전 랜덤 등장으로 시작하고, 문서에는 후속 설계 목차만 둔다.

- 등장 풀: 웨이브, 지역, 타입, 희귀도별 후보군.
- 등장 가중치: 일반/희귀/보스급 포켓몬별 확률.
- 레벨 범위: 웨이브 스케일링에 따른 최소/최대 레벨.
- 중복 보정: 최근 등장한 포켓몬의 반복 출현 억제.
- 도감 보정: 아직 포획하지 못한 포켓몬의 발견 기회 조정.
- MVP 기본값: 1세대 151마리 중 현재 웨이브 레벨 범위에 맞춰 완전 랜덤 선택.

### 회복과 지속 상태

- 일반 웨이브 사이에는 HP를 회복하지 않는다.
- 일반 웨이브 사이에는 주요 이상 상태만 자동 회복한다.
- 포획 성공 포켓몬은 HP 50% 상태로 팀 또는 임시보관에 들어온다.
- 5의 배수 트레이너 배틀 종료 후에는 플레이어 팀 전체 HP를 100% 회복한다.
- 트레이너 배틀 승리 시 전체 회복 후 다음 웨이브로 진입한다.
- 트레이너 배틀 패배 시 실패한 팀 스냅샷을 업로드하고 게임 오버 화면으로 이동한다.
- 트레이너 배틀에서 상대 팀은 저장된 스탯/레벨/스킬을 기반으로 항상 HP 100% 상태로 등장한다.
- 스냅샷에는 현재 HP, 전투 중 버프, 랭크 변화, 이상 상태 같은 일시 상태를 저장하지 않는다.

### 전투강도

전투강도는 포획률, 팀 비교, 트레이너 스냅샷 매칭에 쓰는 단순 점수다.

- `pokemonPower = level * 10 + currentStatTotal + attackMovePower + supportMovePower`
- `currentStatTotal`은 HP를 제외한 현재 전투 스탯 합산값이다.
- `attackMovePower`는 공격기의 위력이며, 고정 데미지 기술은 MVP에서 40점으로 계산한다.
- `supportMovePower`는 보조기의 효과 점수이며, MVP 기본값은 30점이다.
- `teamPower`는 6개 팀 슬롯의 `pokemonPower` 합산값이다.
- 이 점수는 밸런싱용 근사값이며 실제 전투 결과를 대체하지 않는다.

### 포켓볼

MVP 포켓볼은 1세대 151 시리즈에 등장한 포켓볼을 모두 구현한다.

- Poke Ball
- Great Ball
- Ultra Ball
- Master Ball
- Safari Ball

포획률 계산에는 포켓볼별 `pokeballPower`를 사용한다. Master Ball은 MVP에서 포획률 100%로 처리하고, Safari Ball은 Safari Zone 전용이 아니라 일반 포켓볼 상품 중 하나로 취급한다.

## 6. 팀과 보유 포켓몬 설계

### 스타터와 도감 언락

- 첫 접속 유저는 `001`, `004`, `007`을 기본 언락 상태로 시작한다.
- 야생 포켓몬 포획에 한 번이라도 성공하면 해당 종이 도감에 언락된다.
- 새 모험을 시작할 때 유저는 언락된 도감 목록 중 1마리를 스타터로 선택한다.
- 스타터 포켓몬 레벨은 Lv.10으로 고정한다.
- 스타터의 개체 스탯은 게임 규칙에 따라 랜덤 생성한다.
- 스타터의 공격기 1개와 보조기 1개도 현재 기술 선택 규칙에 따라 랜덤 선택한다.

### 팀 슬롯

- 플레이어 팀은 6개 고정 슬롯을 가진다.
- 빈 슬롯이 있으면 포획 성공 포켓몬을 자동으로 가장 앞의 빈 슬롯에 배치한다.
- 팀이 가득 찼으면 포획 성공 포켓몬을 임시보관 상태로 두고 팀 편성 씬에 진입한다.
- 팀 편성 씬에서는 임시보관 포켓몬과 현재 6마리를 한 화면에서 비교하게 한다.
- 유저는 팀 편성 씬에서 기존 슬롯 포켓몬을 유지하거나, 임시보관 포켓몬과 교체한다.

### 보유 목록

- MVP에서는 런 내부 임시보관 목록을 둔다.
- 임시보관 포켓몬은 팀 슬롯이 가득 찬 상태에서 포획에 성공했을 때 들어온다.
- 모험이 끝나면 영구 보관함은 저장하지 않지만, 포획 성공 이력은 도감 언락으로 저장한다.
- 장기 확장에서는 영구 보관함, 도감, 강화, 진화, 해체 보상 등을 추가할 수 있다.

### 상점과 팀 매니징

- 빈 팀 슬롯이 없는 상태에서 포획에 성공하면 전투 후 팀 편성 씬으로 이동한다.
- 상점 페이지에서는 팀 슬롯 포켓몬을 판매하거나 합성할 수 있다.
- 판매/합성으로 슬롯을 비우거나 팀 전투력을 재구성한 뒤 임시보관 포켓몬을 팀에 넣을 수 있다.
- MVP 1차 구현에서는 상점 시스템의 상세 경제 수치는 확정하지 않고, 팀 편성 씬에서 교체만 가능하게 시작한다.

### 비교 UI

포켓몬 카드에는 다음 정보를 항상 표시한다.

- 이름, 도감 번호, 타입, 레벨
- 현재 HP/최대 HP
- 주요 스탯
- 전투강도
- 공격기/보조기
- 팀 슬롯 번호 또는 보유 상태

## 7. 비동기 스냅샷 동기화 설계

### 저장 범위

- 기본 게임 진행 정보는 모두 브라우저 `localStorage`에 저장한다.
- `localStorage`에는 `playerId`, 트레이너 프로필, 언락 도감, 영구 보석, 현재 런 상태, 팀 상태, 상점 상태를 저장한다.
- Google Sheets에는 트레이너전에 필요한 유저 팀 스냅샷과 트레이너전 결과만 저장한다.
- Google Sheets 스냅샷은 포켓몬 ID와 seed로 역산하지 않고, 전투에 필요한 포켓몬의 확정 스탯/레벨/스킬을 그대로 저장한다.

### 백엔드 구조

- Google Apps Script를 웹앱으로 배포한다.
- Google Sheets는 MVP 비동기 상대 풀과 결과 로그 저장소로 사용한다.
- 팀 스냅샷 시트명은 `APT_WAVE_TEAMS`로 한다.
- 트레이너전 결과 시트명은 `APT_TRAINER_RESULTS`로 한다.
- 클라이언트는 Apps Script URL에 HTTP 요청을 보내고, Apps Script가 Sheets를 읽고 쓴다.
- 클라이언트에는 Google Sheets API 키나 OAuth 비밀 정보를 넣지 않는다.

### 트레이너 프로필

- 유저가 첫 게임을 시작할 때 랜덤 트레이너 이미지 스킨을 부여한다.
- 유저는 트레이너 이름을 직접 입력할 수 있다.
- 트레이너 이미지 ID 기본값은 `field-scout`이다.
- 팀 이름 기본값은 트레이너 이름과 동일하다.
- 5의 배수 트레이너 배틀에서 승리하면 팀 스냅샷 기록 화면에서 20자 이내 배틀 인사말을 입력할 수 있다.
- 배틀 인사말은 다른 유저가 해당 스냅샷과 전투를 시작할 때 표시한다.
- 클라이언트는 인사말을 20자로 제한하고, 백엔드는 안전하게 최대 50자까지 잘라 저장한다.

### 저장 시점

클라이언트는 다음 시점에 팀 스냅샷을 전송한다.

- 새 모험 시작 후 초기 팀 확정 시
- 5의 배수 웨이브 트레이너 배틀 종료 시
- 플레이어가 모험을 떠나거나 패배해 런이 종료될 때
- 앱이 닫히기 직전 가능한 경우 `sendBeacon` 또는 동일 목적의 비동기 전송 사용
- 트레이너 배틀에서 패배해 게임 오버가 되더라도 실패한 팀 상태를 업로드한다.

### Sheets 컬럼

`APT_WAVE_TEAMS` 시트는 다음 컬럼을 가진다.

| 컬럼 | 설명 |
| --- | --- |
| `version` | 스냅샷 스키마 버전 |
| `playerId` | 로컬 생성 유저 ID |
| `trainerName` | 유저가 입력한 트레이너 이름 |
| `wave` | 저장 시점의 웨이브 번호 |
| `createdAt` | ISO 문자열 생성 시각 |
| `seed` | 런 또는 전투 시드 |
| `teamPower` | 팀 전체 전투강도 |
| `teamJson` | 6슬롯 팀 상태 JSON |
| `runSummaryJson` | 직전 전투와 런 상태 요약 JSON |
| `trainerPortraitId` | 트레이너 이미지 스킨 ID |
| `teamName` | 팀 이름 |
| `trainerGreeting` | 전투 시작 시 표시할 인사말 |

`APT_TRAINER_RESULTS` 시트는 다음 컬럼을 가진다.

| 컬럼 | 설명 |
| --- | --- |
| `version` | 결과 스키마 버전 |
| `createdAt` | ISO 문자열 생성 시각 |
| `wave` | 트레이너전 웨이브 |
| `seed` | 전투 시드 |
| `attackerPlayerId` | 전투를 진행한 현재 유저 ID |
| `opponentPlayerId` | 상대 스냅샷 소유 유저 ID |
| `opponentSnapshotId` | 상대 팀 스냅샷 ID |
| `winner` | `attacker` 또는 `opponent` |
| `attackerTeamPower` | 현재 유저 팀 전투강도 |
| `opponentTeamPower` | 상대 팀 전투강도 |
| `battleSummaryJson` | 승패, 턴 수, 주요 로그 요약 JSON |

### 팀 스냅샷 JSON

`teamJson`은 현재 HP나 전투 중 상태가 아니라, 대결 상대를 재현하는 데 필요한 확정 전투 정보만 저장한다. 포켓몬은 나중에 상품으로 커스텀될 수 있으므로 `speciesId + seed`로 역산하지 않는다.

```json
{
  "version": 1,
  "slots": [
    {
      "slot": 1,
      "instanceId": "run-pokemon-id",
      "speciesId": "pikachu",
      "speciesNum": 25,
      "name": "Pikachu",
      "level": 12,
      "maxHp": 35,
      "stats": {
        "hp": 35,
        "atk": 22,
        "def": 18,
        "spa": 20,
        "spd": 20,
        "spe": 30
      },
      "selectedMoves": {
        "attack": "thundershock",
        "support": "growl"
      },
      "pokemonPower": 214,
      "customizations": []
    }
  ]
}
```

- 빈 슬롯은 `null`로 저장한다.
- 상대 팀을 전투에 불러올 때 각 포켓몬의 현재 HP는 항상 `maxHp`로 시작한다.
- `customizations`는 상점 상품, 합성, 강화 등으로 생긴 변경 사항을 설명하는 배열이다.
- 언락 도감은 Google Sheets가 아니라 `localStorage`에 저장한다.

### API 계약

Apps Script 웹앱은 같은 배포 URL에서 `doPost(e)`와 `doGet(e)`를 모두 구현한다. `doPost(e)`는 스냅샷 저장과 결과 기록을 처리하고, `doGet(e)`는 같은 웨이브의 상대 스냅샷 랜덤 조회를 처리한다.

#### 스냅샷 저장

`POST {APPS_SCRIPT_URL}?action=submitSnapshot`

요청:

```json
{
  "version": "1",
  "playerId": "local-player-id",
  "trainerName": "그린",
  "wave": 10,
  "createdAt": "2026-05-16T09:00:00.000Z",
  "seed": "battle-seed",
  "teamPower": 1234,
  "teamJson": {
    "version": 1,
    "slots": []
  },
  "runSummaryJson": {
    "type": "trainer",
    "won": true,
    "ended": false
  },
  "trainerPortraitId": "field-scout",
  "teamName": "그린",
  "trainerGreeting": "좋은 승부!"
}
```

응답:

```json
{
  "ok": true
}
```

#### 트레이너전 결과 기록

`POST {APPS_SCRIPT_URL}?action=submitTrainerResult`

요청:

```json
{
  "version": "1",
  "createdAt": "2026-05-16T09:05:00.000Z",
  "wave": 10,
  "seed": "battle-seed",
  "attackerPlayerId": "local-player-id",
  "opponentPlayerId": "opponent-player-id",
  "opponentSnapshotId": "snapshot-id",
  "winner": "attacker",
  "attackerTeamPower": 1234,
  "opponentTeamPower": 1200,
  "battleSummaryJson": {
    "turns": 23,
    "attackerWon": true
  }
}
```

응답:

```json
{
  "ok": true
}
```

#### 상대 스냅샷 조회

상대 스냅샷 조회는 같은 Apps Script의 `doGet(e)`에서 처리한다.

`GET {APPS_SCRIPT_URL}?action=matchSnapshot&wave=10&excludePlayerId=local-player-id&seed=battle-seed`

응답:

```json
{
  "ok": true,
  "snapshot": {
    "snapshotId": "snapshot-id",
    "playerId": "opponent-player-id",
    "trainerName": "레드",
    "wave": 10,
    "teamPower": 1200,
    "teamJson": {
      "slots": []
    },
    "trainerPortraitId": "field-scout",
    "teamName": "레드",
    "trainerGreeting": "덤벼!"
  }
}
```

상대가 없을 때:

```json
{
  "ok": false,
  "reason": "NO_MATCH"
}
```

### 매칭 규칙

- 같은 `playerId`의 스냅샷은 상대 후보에서 제외한다.
- 같은 웨이브 번호의 스냅샷만 후보로 사용한다.
- 같은 웨이브 후보가 여러 개면 랜덤으로 하나를 선택한다.
- 그래도 없으면 로컬 fallback 트레이너를 사용한다.

## 8. UI 설계

### 기본 화면

- 상단: 현재 웨이브, 런 시드, 팀 전투 상태, 자동 진행/일시정지 버튼.
- 중앙: 플레이어 팀 6슬롯과 현재 상대.
- 하단: 전투 로그, 포획 결과, 교체 선택.

### 주요 플로우

- 첫 접속:
  - 기본 언락 포켓몬 `001`, `004`, `007`을 표시
  - 랜덤 트레이너 이미지 스킨 부여
  - 트레이너 이름 입력
- 일반 웨이브 승리 후:
  - 포켓볼 1회 연출
  - 성공/실패 결과 표시
  - 포획 성공 및 빈 슬롯 존재 시 자동 팀 합류
  - 포획 성공 및 빈 슬롯 없음 시 임시보관 후 팀 편성 씬 표시
- 트레이너 배틀 진입 전:
  - 상대 트레이너 카드 표시
  - 상대 트레이너 인사말 표시
  - 상대가 실제 다른 유저 스냅샷인지 fallback인지 표시
- 트레이너 배틀 종료 후:
  - 승리 시 팀 스냅샷 기록 화면에서 20자 이내 인사말 입력
  - 승리 시 팀 전체 HP 100% 회복 후 다음 웨이브 진입
  - 패배 시 실패한 팀 스냅샷 업로드 후 게임 오버 화면 표시
- 팀 교체:
  - 현재 6슬롯과 새 포켓몬을 한 화면에 표시
  - 교체할 슬롯 선택 또는 기존 팀 유지 선택
  - 상점 확장 후에는 판매/합성으로 슬롯 관리

## 9. 테스트 계획

- 데이터 검증
  - 1세대 151마리 이미지가 존재해야 한다.
  - 모든 포켓몬은 공격기/보조기 선택 또는 fallback이 가능해야 한다.
- 전투 검증
  - 같은 시드와 같은 팀 스냅샷은 같은 전투 로그와 결과를 만든다.
  - 6슬롯 팀 전투에서 쓰러진 포켓몬 다음 슬롯이 자동 출전한다.
  - 한쪽 팀이 모두 쓰러지면 승패가 확정된다.
  - 일반 웨이브 사이에는 HP가 유지되고 주요 이상 상태만 회복된다.
  - 트레이너 배틀 승리 후에는 팀 전체 HP가 100% 회복된다.
- 웨이브 검증
  - `wave % 5 !== 0`에서는 야생 포켓몬 전투가 발생한다.
  - `wave % 5 === 0`에서는 트레이너 스냅샷 전투가 발생한다.
  - 트레이너 스냅샷 조회 실패 시 fallback 상대가 생성된다.
- 포획 검증
  - 야생 포켓몬 HP가 0이 된 뒤 1회만 포획 판정이 발생한다.
  - 트레이너 배틀에서는 포획이 발생하지 않는다.
  - 포획 성공 포켓몬은 HP 50% 상태로 들어온다.
  - 빈 슬롯이 있으면 포획 성공 포켓몬이 자동 팀 합류한다.
  - 빈 슬롯이 없으면 포획 성공 포켓몬이 임시보관되고 팀 편성 씬이 열린다.
  - 같은 시드는 같은 포획 결과를 만든다.
- 도감/스타터 검증
  - 첫 접속 유저는 `001`, `004`, `007`이 언락되어 있다.
  - 포획 성공한 포켓몬은 다음 런부터 스타터 후보로 표시된다.
  - 스타터 포켓몬은 Lv.10으로 생성된다.
- 동기화 검증
  - Apps Script 저장 API가 `APT_WAVE_TEAMS` 시트에 지정 컬럼 순서로 스냅샷을 추가한다.
  - 매칭 API가 같은 `playerId`를 제외하고 상대 스냅샷을 반환한다.
  - 트레이너 배틀 패배 팀도 스냅샷으로 업로드된다.
  - 트레이너 이름, 초상화 ID, 팀 이름, 인사말이 저장되고 조회된다.
  - 클라이언트 인사말은 20자 이내로 제한된다.
  - API 실패 시 게임 진행이 막히지 않는다.
- UI 검증
  - 모바일 폭 360px에서 6슬롯 팀, 전투 로그, 포획/교체 UI가 겹치지 않는다.
  - 트레이너 이름 입력, 인사말 입력, 상대 인사말 표시가 모바일에서 잘리지 않는다.
  - GitHub Pages base path에서 이미지와 정적 리소스가 로드된다.
- 빌드 검증
  - `npm run test`가 통과해야 한다.
  - `npm run build`가 통과해야 한다.
